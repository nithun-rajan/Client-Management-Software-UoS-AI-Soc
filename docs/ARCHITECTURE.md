## Overview

The Estate Agent CRM is a production-grade, full-stack application built with modern DevOps and SRE practices. It provides comprehensive property management, landlord tracking, and tenant applicant management with enterprise-level observability, security, and reliability.

### Design Principles

1. **Security First**: Defense in depth with multiple security layers
2. **Observable**: Comprehensive monitoring, logging, and tracing
3. **Reliable**: SLO-driven reliability with error budget tracking
4. **Scalable**: Horizontal scaling with stateless architecture
5. **Maintainable**: Clean code, comprehensive tests, clear documentation

## Design Patterns

### Patterns Used

1. **Repository Pattern** (Data Access)
   - Abstracts database access
   - Enables testing with mock repositories
   - Centralizes query logic

2. **Dependency Injection** (Framework-level)
   - FastAPI's dependency system
   - Database session management
   - Configuration injection

3. **Middleware Chain** (Cross-cutting Concerns)
   - Security, logging, metrics
   - Separation of concerns
   - Ordered execution

4. **DTO Pattern** (Data Transfer)
   - Pydantic models for API contracts
   - Clear separation from ORM models
   - Validation at boundaries

5. **Circuit Breaker** (Resilience)
   - Protect against cascading failures
   - Fail fast when dependencies down

6. **Observer Pattern** (Events)
   - Event logging system
   - Audit trail
   - Decoupled components
